# 性能优化

高并发场景
## 当数据库受到高并发访问时，可以使用以下方案：
- 使用消息队列缓存写入的数据，将并发写入变成异步写入。
- 数据库采用主从架构，实现数据备份、读写分离。

- 还可采用以下方法提高查询效率：
  - 将数据库尽量分表、分库（比如不同业务的分开），减少每张表的大小。
  - 正确地设置索引。
  - 在用户与数据库之间增加一个cache层，缓存用户经常请求的数据。
